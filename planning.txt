Votting app functionality

1. User sign in / signup
2. User can see the list of candidate
3. Vote one of the candidate, after voting user can not vote again
4. there is  a route which shows the list of the candidates and their live vote counts sorted by their vote counts
5. user data must contain their one unique government id named: Aadhar card no.
6. there should be one admin who can only maintane the table of candidates and he cann't able to vote at all
7. user can change their password
8. user can only login with Aadhar card no. and password


=================================================================

Routes 

User Authentication:
    /signup: POST - Create a new user account
    /login:  POST - login to an existing account [Aadhar card no. + password]

Voting:
    /candidates: GET - Get the list of the candidates
    /vote/candidateId: POST - Vote for a specific candidate

Vote Counts:
    /vote/counts: GET - Get the list of candidates sorted by their vote counts.


User Profile: 
    /profile: GET - Get the user's profile information
    /profile/password: PUT - change the user's password

Admin Candidate Management:
    /candidates: POST - Create a new candidate
    /candidates/candidateId: PUT - Update an existing candidate
    /candidates/candidateId: DELETE - Delete a candidate from the list
